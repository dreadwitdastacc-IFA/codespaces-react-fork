🎯 Complex CSS Selectors

Understanding Specificity with Real-World Examples

Example 1: ID + Class + Element

#myElement input.myClass
1-1-1
Components:
  • 1 ID selector (#myElement)
  • 1 Class selector (.myClass)
  • 1 Element selector (input)
Power: Very high specificity - difficult to override

Example 2: Attribute + Pseudo-class + Element

input[type="password"]:required
0-2-1
Components:
  • 1 Attribute selector ([type="password"])
  • 1 Pseudo-class (:required)
  • 1 Element selector (input)
Power: Medium specificity - weaker than IDs but stronger than single classes

Example 3: Attribute Selector vs ID

[id="myElement"] input.myClass
0-2-1
vs
#myElement input.myClass
1-1-1
Key Difference: Using attribute selector [id="..."] vs ID selector #

Winner: #myElement (1-1-1) beats [id=""] (0-2-1)

Rule: ID selectors always win in the first column of specificity

Example 4: Multiple Element Selectors

html body .container main input
0-1-4
Components:
  • 4 Element selectors (html, body, .container, main)
  • 1 Class selector (.container)
Key Insight: Even 4 elements combined cannot beat 1 ID selector!

Example 5: :not() Pseudo-class

input:not(#fakeId)
1-0-1
Components:
  • 1 ID inside :not() (#fakeId) - counts!
  • 1 Element selector (input)
Key Rule: :not() arguments contribute to specificity calculation

Example 6: :is() Pseudo-class

input:is(.required-field, #special)
1-0-1
Components:
  • :is() takes HIGHEST specificity from arguments
  • #special (1-0-0) wins over .required-field (0-1-0)
  • 1 Element selector (input)
Key Rule: :is() uses the most specific argument's weight

✨ Key Takeaways

  • IDs always win in the first column - one ID beats any combination of classes
  • Attribute selectors count as classes (0-1-0), same as .class
  • Pseudo-classes like :required also count as classes (0-1-0)
  • :not() arguments contribute their specificity weight
  • :is() takes the highest specificity from its arguments
  • :where() has zero specificity - ignores all arguments
  • Multiple elements (html body main) still can't beat a single ID
  • Cascade order matters only when specificity is equal
